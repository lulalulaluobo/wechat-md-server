# wechat-md-server 产品需求文档 (PRD)

> 版本：V5 | 最后更新：2026-05-05

---

## 1. 产品定位

### 1.1 一句话定义

面向 Obsidian 用户的自托管本地服务，将微信公众号文章和普通网页长文自动抓取、清洗、转换为结构化 Markdown，并同步到 Obsidian 笔记库。

### 1.2 目标用户

| 用户画像 | 核心诉求 |
|---------|---------|
| Obsidian 重度用户 | 看到好文章想快速入库，建立个人知识库 |
| 信息工作者 | 需要批量归档公众号文章用于后续检索和整理 |
| 知识管理爱好者 | 希望文章入库时自带 AI 生成的摘要、标签、Frontmatter |
| 团队/小型组织 | 通过 Bot 接口共享文章转换能力 |

### 1.3 解决的核心问题

1. **公众号文章难以长期保存**：微信文章 URL 会失效、阅读体验受限于微信客户端
2. **手动复制粘贴效率低**：格式丢失、图片无法保留、批量操作不现实
3. **Obsidian 入库流程断裂**：转换后的 Markdown 需要手动拖入 Vault，无法形成自动化闭环

### 1.4 产品边界

- **做**：微信公众号文章 + 普通网页文章 → Markdown → Obsidian 同步
- **不做**：不是通用爬虫平台，不做视频/音频转换，不做知识图谱，不做笔记编辑器

## 3. 功能需求

### 3.1 用户认证与权限

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 登录/登出 | 用户名 + 密码登录，Cookie + CSRF Token 双重校验 | P0 |
| 密码安全 | PBKDF2-SHA256 哈希（200,000 次迭代），不明文存储 | P0 |
| 登录保护 | 同 IP+账号 10 分钟内 5 次失败则锁定 15 分钟 | P0 |
| 角色控制 | admin / operator 两种角色，管理操作限 admin | P1 |
| 审计日志 | 记录管理操作的执行人、IP、操作详情 | P1 |
| 密码重置 | CLI 命令行重置密码（支持指定或随机生成） | P1 |

### 3.2 文章转换

#### 3.2.1 来源支持

| 来源类型 | 描述 | 优先级 |
|----------|------|--------|
| 微信公众号文章 | 直接抓取 `mp.weixin.qq.com` 页面 HTML 并解析 | P0 |
| 普通网页文章 | 使用 readability + BeautifulSoup 提取正文 | P0 |
| 知乎 | 明确拒绝并提示用户（技术限制） | P1 |

#### 3.2.2 转换能力

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 单篇转换 | 粘贴一个 URL，同步返回转换结果 | P0 |
| 批量转换 | 多个 URL 排队处理，返回 batch job ID | P0 |
| 文件导入 | 上传 .txt 文件批量导入 URL 列表 | P1 |
| 子进程隔离 | 单篇转换在独立子进程中执行，带超时保护 | P0 |

#### 3.2.3 转换 Pipeline

```
URL 输入 → 来源识别 → HTML 抓取（带缓存）→ HTML 清洗 → 图片处理 →
正文提取 → HTML→Markdown 转换 → 后处理（格式规范化）→ AI 润色（可选）→
输出 Markdown 文件 → 同步到 Obsidian
```

各环节详细要求：

| 环节 | 要求 |
|------|------|
| 来源识别 | 根据 URL 域名自动判断 wechat / web / zhihu |
| HTML 抓取 | 伪装浏览器 User-Agent，支持磁盘缓存（SHA-256 索引） |
| HTML 清洗 | 移除 script/style/comment，解析 lazy-load 图片（data-src → src），移除推广/引流/加群等噪音内容 |
| 图片处理 | 见 3.3 节 |
| Markdown 转换 | 自定义 HTMLParser，支持标题/列表/表格/代码块/引用/行内格式 |
| 后处理 | 空行规范化、代码块修复、表格分隔符补全 |
| 输出 | 自动编号目录结构（001_Title/、002_Title/...） |

### 3.3 图片处理

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| `wechat_hotlink` | 保留微信 CDN 原始链接 | 快速上手，不额外依赖存储服务 |
| `s3_hotlink` | 下载图片 → 可选缩放（最大 1600px）→ 转 WebP（质量 78）→ 上传 S3 兼容存储 | 图片长期可控、可迁移 |

**S3 模式配置项**：Endpoint、Bucket、Access Key、Secret Key、Region、Public URL 前缀

### 3.4 Obsidian 同步

| 功能 | 描述 | 优先级 |
|------|------|--------|
| FNS 连接配置 | 配置 Flying Server 地址、API Token、Vault 名称、目标目录 | P0 |
| 连通性测试 | 保存配置前验证 FNS 是否可达 | P0 |
| 默认目标目录 | `00_Inbox/微信公众号` | P0 |
| 自动同步 | 转换成功后自动调用 FNS API 同步文件 | P0 |

### 3.5 AI 润色

#### 3.5.1 Provider 支持

| Provider | 说明 |
|----------|------|
| OpenAI Compatible | 任何兼容 OpenAI API 格式的端点 |
| Anthropic | Claude 系列 |
| Gemini | Google Gemini |
| Ollama | 本地部署的 Ollama 模型 |
| OpenRouter | OpenRouter 聚合网关 |

#### 3.5.2 润色能力

| 输出项 | 描述 | 优先级 |
|--------|------|--------|

| tags | 文章标签（用于 Obsidian 分类检索） | P1 |

| body_polish | 正文 AI 润色块（补充理解、点评等） | P2 |

#### 3.5.3 模板系统

| 功能 | 描述 |
|------|------|
| Prompt 模板 | 支持 `{{variable}}` 占位符的自定义提示词模板 |
| Frontmatter 模板 | 自定义 YAML 输出格式 |
| Body 模板 | 自定义正文润色块的插入格式 |
| 模板来源 | `manual`（手写）和 `clipper_import`（从外部工具导入） |

### 3.6 微信公众号自动同步

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 公众号搜索 | 按关键词搜索公众号，查看账号信息 | P1 |
| 同步源管理 | 添加/删除要持续关注的公众号 | P1 |
| QR 码登录 | 通过微信公众号后台扫码登录获取凭据 | P1 |
| 定时同步 | 后台定时拉取新文章列表 | P1 |
| 文章列表 | 分页浏览已同步的文章，按状态筛选 | P1 |
| 批量入库 | 选择多篇文章批量触发转换 + 同步 | P1 |
| 调度配置 | 可配置同步频率、时区、暂停/恢复 | P2 |

### 3.7 任务管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 任务列表 | 查看所有转换任务，支持按状态/来源/触发方式筛选 | P0 |
| 任务详情 | 查看任务状态、来源类型、触发方式、标题、原始链接 | P0 |
| 失败重跑 | 单篇或批量重跑失败任务 | P0 |

### 3.8 Bot 集成

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Telegram Bot | 接收用户发送的文章 URL，异步转换后回复结果 | P1 |
| 飞书 Bot | 接收用户发送的文章 URL，异步转换后回复结果 | P1 |
| Webhook 入口 | 统一的 POST webhook 端点接收消息 | P1 |

### 3.9 系统设置

| 配置分类 | 配置项 |
|---------|--------|
| FNS 同步 | 服务地址、API Token、Vault 名称、目标目录 |
| 图片模式 | wechat_hotlink / s3_hotlink |
| S3 配置 | Endpoint、Bucket、Access Key、Secret Key、Region、Public URL |
| AI 配置 | Provider 类型、Endpoint、API Key、模型名称、模板 |
| 调度配置 | 同步频率、时区、暂停时间 |

**安全要求**：所有敏感配置（API Key、Token、密码）使用 Fernet 加密存储，加密密钥来自 `WECHAT_MD_APP_MASTER_KEY` 环境变量。

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|------|------|
| 单篇文章转换 | ≤ 30 秒（不含 AI 润色） |
| 批量转换 | 支持至少 50 篇排队处理 |
| 并发 | 单实例部署，子进程隔离保证不互相阻塞 |

### 4.2 安全

| 要求 | 实现方式 |
|------|---------|
| 密码安全 | PBKDF2-SHA256，200,000 次迭代 |
| 会话安全 | HMAC 签名 Token + CSRF Double Cookie |
| 敏感数据加密 | Fernet 对称加密（密钥由环境变量注入） |
| 登录防暴力破解 | 10 分钟内 5 次失败锁定 15 分钟 |
| 安全响应头 | X-Frame-Options / X-Content-Type-Options / Referrer-Policy |
| 生产 Cookie | SESSION_COOKIE_SECURE=true |

### 4.3 可靠性

| 要求 | 说明 |
|------|------|
| 数据持久化 | SQLite WAL 模式 + JSONL 任务日志 + 磁盘缓存，Docker Volume 挂载 |
| 任务可追溯 | 每次转换生成 execution 记录，记录触发渠道、状态、AI 使用情况 |
| 失败可重试 | 任何失败任务支持一键重跑 |
| 缓存去重 | 基于内容 SHA-256 的磁盘缓存，相同 URL 不重复抓取 |

### 4.4 部署与运维

| 要求 | 说明 |
|------|------|
| Docker 一键部署 | docker compose up -d 启动全部服务 |
| 一键安装脚本 | Debian/Ubuntu 一键安装/更新/卸载 |
| 健康检查 | Docker health check 探针 |
| 数据目录 | 所有运行时数据集中到 /app/data/，通过 Volume 持久化 |
| 非 root 运行 | Docker 容器以非 root 用户运行 |

### 4.5 可测试性

| 测试类型 | 覆盖范围 |
|---------|---------|
| 单元测试 | Pipeline、AI 润色、内容源、任务历史、配置、认证 |
| 集成测试 | API 端点、FNS 同步、进程隔离 |
| 功能回归 | V5.2 特性回归测试 |
| 构建验证 | Docker 构建产物验证 |

## 5. 技术架构

### 5.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| 语言 | Python 3.14 |
| Web 框架 | FastAPI + Uvicorn |
| 数据库 | SQLite3 (WAL 模式) |
| 前端 | 原生 HTML/CSS/JS（无框架） |
| 认证 | PBKDF2 + HMAC + Fernet |
| HTTP 客户端 | requests / httpx |
| HTML 解析 | BeautifulSoup4 + readability-lxml |
| 图片处理 | Pillow（缩放/WebP 转换） |
| AI 集成 | 多 Provider 适配器模式 |
| 容器化 | Docker + Docker Compose |

## 6. 页面与交互

### 6.1 页面清单

| 页面 | 路由 | 核心功能 |
|------|------|---------|
| 登录 | `/login` | 用户名密码登录 |
| 转换中心 | `/` | 单篇/批量 URL 转换、文件导入 |
| 设置 | `/settings` | FNS 配置、图片模式、AI 配置、调度管理 |
| 同步管理 | `/sync` | 公众号搜索、同步源管理、QR 码登录 |
| 文章列表 | `/articles` | 已同步文章浏览、筛选、批量入库 |

### 6.2 关键交互流程

**单篇转换流程**：
```
用户粘贴 URL → 点击转换 → 显示进度 → 转换完成 → 提示同步结果
```

**批量转换流程**：
```
用户粘贴多个 URL 或上传文件 → 提交 → 后台排队 → 轮询任务状态 → 全部完成汇总展示
```

**公众号自动同步流程**：
```
扫码登录微信后台 → 搜索公众号 → 添加为同步源 → 配置同步频率 →
后台定时拉取新文章 → 用户在文章列表选择入库 → 触发转换+同步
```

## 9. 约束与风险

| 类别 | 描述 |
|------|------|
| 内容版权 | 用户需自行确保使用合法合规，尊重原创版权 |
| 微信反爬 | 公众号文章抓取依赖当前 HTML 结构，微信改版可能导致失败 |
| 知乎限制 | 明确不支持知乎链接 |
| 图片链接时效 | wechat_hotlink 模式下微信 CDN 链接可能过期 |
| 单实例 | 当前不支持多实例水平扩展 |
| 存储依赖 | SQLite 单写，高并发场景可能成为瓶颈 |
